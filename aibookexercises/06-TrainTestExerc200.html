Step 7 - Tune the Workflow and Train All Models:

Now it is time to run the tuning procedure using the tune_grid() command. Be patient because it will take some time to fully execute. Since we have to try out 10 parameters and use five folds for each model, the tune_grid() command has to fit 50 models \((10\cdot 5=50)\).

Please substitute ______ with a list of metrics to be calculated. Use the metrics rmse (Root Mean Squared Error), mae (Mean Absolute Error) and rsq (r Squared \(r^2\)).

After the tuning is finished, all results are stored in the R object TuneResults, and they can be evaluated by different metrics commands.

For example the command autoplot() provides a diagrammatic overview of the results for the three metrics.

Here is a hint:

The command metric_set() takes a comma-separated list of metrics to be evaluated.

Consider that you want to evaluate the metrics rmse, mae, and rsq.

Here is the Solution:

TuneResults=tune_grid(TuneWFModelPoly, resamples=FoldsForTuning, grid=ParGrid, metrics=metric_set(rmse, mae, rsq)) autoplot(TuneResults) 
TuneResults=tune_grid(TuneWFModelPoly, resamples=FoldsForTuning,
                      grid=ParGrid, metrics=metric_set(rmse, mae, rsq))
autoplot(TuneResults)

The three graphs that you created in the exercise above show the values for each rmse, mae, and rsq for all tried out hyper-parameters (degree).

To calculate these values the five folds that were used for validation are averaged.

It looks like \(r^2\) has the clearst fluctutions. It seems reasonable to assume that degree=6 provides the best results. Let us confirm this in the Step 8.
